# Roadmap

## Phase 1 — Research consolidation

- Map existing fact-checking systems.
- Identify benchmarks.
- Define claim schema.
- Define scoring logic.

## Phase 2 — MVP

- Build FastAPI backend.
- Add claim extraction.
- Add evidence retrieval.
- Add basic verification.
- Add confidence score.

## Phase 3 — Evaluation

- Test on FEVER / SciFact-style claims.
- Measure errors.
- Improve retrieval.
- Improve scoring.

## Phase 4 — Decentralized design

- Define miner role.
- Define validator role.
- Design reward mechanism.
- Study collusion and Sybil attacks.

## Phase 5 — Public prototype

- Demo interface.
- Documentation.
- Contributor onboarding.
